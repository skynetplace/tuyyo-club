## 2. ТЕХНОЛОГІЧНИЙ СТЕК (ARCHITECTURE OVERVIEW)
Ми відмовляємося від важких SPA (React/Vue) на користь **гіперлегкої архітектури**, яка гарантує 100/100 балів у Google PageSpeed.

*   **Frontend (Public UI):** Чистий **HTML5**, **Bootstrap 5.3** (CSS + мінімальний JS), **HTMX 2.0** (для SPA-взаємодії без перезавантажень).
*   **Backend (API & Edge Logic):** **Cloudflare Workers** (мікро-фреймворк **Hono**).
*   **Database:** **Cloudflare D1** (Edge SQLite).
*   **Media Storage:** **Cloudflare R2** (S3-сумісне сховище для фото/відео/PDF).
*   **Hosting & CDN:** **Cloudflare Pages** (Frontend) + Global Edge Network.
*   **Admin Auth:** **Cloudflare Zero Trust** (захист адмінки без написання власного логіна).
*   **Payments:** Stripe Checkout (Redirect flow), Bizum, Cash.
*   **Notifications:** Resend (Email), WhatsApp Cloud API, Telegram Bot API.

## 4. БАЗА ДАНИХ (CLOUDFLARE D1 SCHEMA)
Нормалізована SQLite схема, оптимізована для Edge-виконання та підтримки CMS.

```sql
-- 1. Локації (Зони доставки)
CREATE TABLE locations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL UNIQUE, lat REAL NOT NULL, lng REAL NOT NULL,
    delivery_fee REAL DEFAULT 20.0, is_active BOOLEAN DEFAULT 1
);

-- 2. Інвентар (SUP-борди та інше оснащення)
CREATE TABLE inventory (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    model_name TEXT NOT NULL, max_weight_kg INTEGER,
    price_2h REAL, price_half_day REAL, price_full_day REAL,
    deposit_amount REAL DEFAULT 150.0,
    total_units INTEGER DEFAULT 5, -- 🔥 КРИТИЧНО: Загальна кількість для Anti-Overbooking
    image_url TEXT, is_active BOOLEAN DEFAULT 1
);

-- 3. Клієнти
CREATE TABLE customers (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    full_name TEXT NOT NULL, phone TEXT NOT NULL, email TEXT,
    id_document TEXT, created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 4. Бронювання
CREATE TABLE bookings (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    customer_id INTEGER NOT NULL, location_id INTEGER NOT NULL,
    start_time DATETIME NOT NULL, end_time DATETIME NOT NULL,
    total_amount REAL NOT NULL, payment_method TEXT DEFAULT 'stripe',
    payment_status TEXT DEFAULT 'unpaid', 
    stripe_session_id TEXT UNIQUE,
    legal_agreement BOOLEAN DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 5. Позиції бронювання
CREATE TABLE booking_items (
    booking_id INTEGER NOT NULL, inventory_id INTEGER NOT NULL,
    quantity INTEGER NOT NULL, price_at_booking REAL NOT NULL,
    PRIMARY KEY (booking_id, inventory_id)
);

-- 6. Відгуки
CREATE TABLE reviews (
    id INTEGER PRIMARY KEY AUTOINCREMENT, booking_id INTEGER UNIQUE,
    customer_name TEXT, rating INTEGER, comment TEXT, photo_url TEXT
);

-- 7. Глобальні налаштування та Інтеграції (Для Адмінки)
CREATE TABLE settings (
    key TEXT PRIMARY KEY, value TEXT NOT NULL, is_secret BOOLEAN DEFAULT 0
);
INSERT INTO settings (key, value) VALUES 
('payment_stripe_enabled', '1'), ('payment_bizum_enabled', '0'), ('payment_cash_enabled', '1'),
('notify_telegram_enabled', '0'), ('notify_email_enabled', '1'), ('notify_whatsapp_enabled', '0');

-- 8. Багатомовність
CREATE TABLE translations (
    lang_code TEXT NOT NULL, translation_key TEXT NOT NULL, translation_value TEXT NOT NULL,
    PRIMARY KEY (lang_code, translation_key)
);

-- 9. Юридичні документи
CREATE TABLE documents (
    id INTEGER PRIMARY KEY AUTOINCREMENT, doc_type TEXT NOT NULL, lang_code TEXT NOT NULL,
    title TEXT NOT NULL, content_html TEXT, pdf_url TEXT, is_active BOOLEAN DEFAULT 1
);

-- ІНДЕКСИ (Критично для швидкості)
CREATE INDEX idx_bookings_time ON bookings(start_time, end_time, payment_status);
CREATE INDEX idx_items_inv ON booking_items(inventory_id);
```

## 5. BACKEND API (CLOUDFLARE WORKERS + HONO)
Worker повертає **готові HTML-фрагменти** для HTMX, що робить фронтенд неймовірно легким.

### 5.1. Обробка запитів та Race Conditions
```javascript
import { Hono } from 'hono';
import { cors } from 'hono/cors';

const app = new Hono();

// Жорсткий CORS
app.use('/api/*', cors({ origin: ['https://tuyyo.com', 'https://www.tuyyo.com'] }));

// Перевірка доступності (Повертає HTML для HTMX)
app.get('/api/check-availability', async (c) => {
  const db = c.env.DB;
  const { date, inventory_id } = c.req.query();
  
  const booked = await db.prepare(`
    SELECT SUM(bi.quantity) as total_booked FROM booking_items bi
    JOIN bookings b ON bi.booking_id = b.id
    WHERE bi.inventory_id = ? AND b.start_time LIKE ? AND b.payment_status != 'failed'
  `).bind(inventory_id, `%${date}%`).first();

  const inv = await db.prepare("SELECT total_units FROM inventory WHERE id = ?").bind(inventory_id).first();
  const available = inv.total_units - (booked.total_booked || 0);
  
  return c.html(`<div class="alert alert-${available > 0 ? 'success' : 'danger'} py-2">Доступно: <strong>${available}</strong></div>`);
});

// Створення бронювання (Захист від Race Conditions)
app.post('/api/create-booking', async (c) => {
  const db = c.env.DB;
  await db.prepare("BEGIN IMMEDIATE").run(); // Блокуємо запис
  try {
    // ... перевірка доступності, інсерт в bookings та booking_items ...
    await db.prepare("COMMIT").run();
    // ... генерація Stripe Session або збереження для Cash/Bizum ...
    return c.json({ success: true, redirect_url: stripeSession.url });
  } catch (e) {
    await db.prepare("ROLLBACK").run();
    return c.json({ error: "Booking failed" }, 500);
  }
});

export default app;
```

### 6.1. Multi-Step Booking Form (Серце конверсії)
Форма бронювання реалізована як один контейнер, який HTMX динамічно перезаповнює.

```html
<!-- Крок 1: Дата та Локація -->
<form id="booking-wizard" hx-post="/api/booking/step-2" hx-target="#booking-wizard" hx-swap="innerHTML">
  <div class="mb-3">
    <label class="form-label">Оберіть дату</label>
    <input type="date" name="date" id="date-input" class="form-control" required>
    <!-- HTMX: Автоматично перевіряє наявність при зміні дати -->
    <div hx-get="/api/check-availability" hx-trigger="change from:#date-input" hx-include="[name='inventory_id']" id="availability-msg"></div>
  </div>
  <button type="submit" class="btn btn-primary w-100">Далі: Вибір дошок</button>
</form>
```

### 6.2. SEO та Core Web Vitals (Target: 100/100)
*   **LCP:** Hero-зображення інлайниться в перші 14КБ HTML або має `fetchpriority="high"`.
*   **CLS:** Усі зображення мають жорсткі `aspect-ratio`.
*   **INP:** Відсутність важкого JS. HTMX (14KB) + Bootstrap JS (60KB) — це максимум, що завантажується.
*   **Meta-теги:** Динамічна генерація Open Graph через Worker для сторінок відгуків/акцій.

1.  **Інвентар та Ціни:** Додавання/редагування дошок. Поле `total_units` гарантує, що клієнти не забронюють більше, ніж є фізично. Гнучке управління цінами.

3.  **Медіа-менеджер:** Drag-and-drop завантаження фото/відео/PDF прямо у **Cloudflare R2**.
4.  **Опціональні Інтеграції (Toggles):**
    *   *Платежі:* Тогли [ON/OFF] для Stripe, Bizum, Cash. Якщо Stripe ON — з'являються поля для API Keys (зберігаються в D1 з `is_secret=1`).
    *   *Сповіщення:* Тогли для Telegram (Bot Token), Email (Resend Key), WhatsApp.
5.  **Юридичні Документи:** Завантаження PDF або написання HTML-тексту для "Правил безпеки", "Договору оренди". Прив'язка до мови (EN/ES).

## 9. БІЗНЕС-ЛОГІКА ТА ІНТЕГРАЦІЇ

### 9.1. Payment Flow (Stripe)
1. Клієнт заповнює форму (HTML/HTMX).
2. Натискає "Оплатити" -> Worker створює запис у D1 (`payment_status = 'unpaid'`).
3. Worker генерує Stripe Checkout Session і повертає URL.
4. HTMX робить `window.location.href = url` (редирект на Stripe).
5. Після оплати Stripe шле Webhook на `/api/webhooks/stripe`.
6. Worker оновлює статус на `paid` і тригерить відправку Email/WhatsApp клієнту та Telegram власнику.

### 9.2. Альтернативні оплати (Bizum / Cash)
Якщо адмін вимкнув Stripe, на фінальному кроці форми HTMX запитує `/api/booking/render-payment`. Worker бачить, що Stripe OFF, і рендерить кнопку "Підтвердити бронювання (Оплата на місці)" та реквізити для Bizum.
